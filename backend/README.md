# Water Tank Monitoring System

A comprehensive, real-time 24/7 water tank monitoring solution built with Python, Node.js, React, and MongoDB. This system interfaces with ESP32-based pressure sensors to continuously track water levels, evaluate thresholds, generate alerts, and provide a full-featured admin dashboard for management and live log viewing.

## 🌟 Features

* **Continuous Monitoring:** A dedicated Python background service that fetches voltage and pressure readings from ESP32 devices on your local network.
* **Real-time Logs:** View live operational logs in the browser streamed via WebSockets, with features to pause, filter by severity, search, and clear logs.
* **Service Control:** Start, stop, and monitor the health of the Python monitoring service directly from the web dashboard.
* **Configuration Management:** Manage your ESP32 devices (Names, Unit IDs, and IP Addresses) seamlessly through the UI.
* **Dynamic Settings:** Update global parameters like the MongoDB Connection URI, fetch intervals, and low-water alert thresholds without touching the code.
* **Graceful Shutdown:** Safely terminate the backend, frontend, and background processes with a single click.

## 🏗️ Architecture

The system is divided into three main components:

1. **Monitor Service (Python):**
   * Connects to configured ESP32 IPs via HTTP.
   * Calculates water pressure and estimated level percentages.
   * Logs data and triggers alerts into a MongoDB database.
   * Script: `monitor_service.py`

2. **Backend API & WebSocket Server (Node.js/Express):**
   * Acts as the bridge between the UI, the Python service, and the log files.
   * Spawns and manages the Python background process.
   * Streams `water_monitor.log` updates to the frontend via WebSockets.
   * Serves endpoints for configuration and settings management.
   * Directory: `gui-backend/`

3. **Admin Dashboard (React/Vite):**
   * A modern, responsive single-page application built with React and Tailwind CSS.
   * Provides pages for Dashboard, Config Manager, Logs Viewer, Alerts, Service Control, and Settings.
   * Directory: `gui-frontend/`

## 🚀 Prerequisites

Before you begin, ensure you have the following installed on your system (Windows recommended):

* [Python 3.8+](https://www.python.org/downloads/) (Make sure it is added to your PATH)
* [Node.js 18+](https://nodejs.org/) (Includes npm)
* A running MongoDB instance or a MongoDB Atlas Cluster URI.

## 🛠️ Quick Start (Windows)

The easiest way to get everything up and running is by using the provided startup script:

1. Clone this repository to your local machine.
2. Open the project folder.
3. Double-click the `start.bat` file.

**What `start.bat` does:**
* Checks for Python and Node.js installations.
* Installs required Python packages (`requests`, `pymongo`).
* Installs Node.js dependencies for both the backend and frontend.
* Starts the Node.js backend on port `3001` in the background.
* Starts the Vite frontend server on port `5173` in the background.
* Automatically opens your default web browser to `http://localhost:5173/`.

> **Note:** Keep the command prompt window opened by `start.bat` running. It acts as the master process tree. You can shut down the entire system via the "Shutdown System" button in the web dashboard, which will cleanly kill all processes and close the terminal.

## ⚙️ Manual Setup & Running

If you prefer to start the services manually or are running on a non-Windows OS:

### 1. Setup Python Dependencies
```bash
pip install -r requirements.txt
# OR
pip install requests pymongo
```

### 2. Start the Backend
```bash
cd gui-backend
npm install
node server.js
```
The backend will run on `http://localhost:3001`.

### 3. Start the Frontend
Open a new terminal window:
```bash
cd gui-frontend
npm install
npm run dev -- --port 5173
```
The frontend will be accessible at `http://localhost:5173`.

## 🌐 Configuring MongoDB

On the first run, the system uses a default MongoDB URI defined in `monitor_service.py` or `.env`. 

To change the database connection:
1. Open the Web Dashboard.
2. Navigate to the **Settings** page.
3. Enter your new **MongoDB Connection String**.
4. Click **Save Changes**.
5. Go to the **Service Control** page and **Restart** the Python Monitor Service for the changes to take effect.

## 📝 Logging

The Python service utilizes standard Python logging to write operational details, errors, and system states to `logs/water_monitor.log`. 

The Node.js backend actively polls this file and broadcasts new lines to the React frontend over WebSockets, allowing you to monitor the system's health in true real-time from the **Logs Viewer** tab.
